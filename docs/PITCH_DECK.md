## 🎯 1. 核心定位与愿景 (Product Vision)

### 1.1 一句话电梯演讲 (Elevator Pitch)
“Ocean 不是一个套壳聊天的问答机器人，而是一个专为金融级内网研发效能痛点打造的**行动导向型智能体中枢 (Action-Oriented Agent Hub)**。它通过原生 **MCP (Model Context Protocol)** 协议打通了代码库、流水线和项目管理系统，并在业内首创了 **AGP 治理协议** 以保障金融级安全。开发者用自然语言即可驱动机器完成‘排查异常-建Bug-合代码-发版’的全自动化流转。”

### 1.2 核心解决的痛点 (Pain Points)
1. **系统孤岛壁垒**：开发人员每天需要在 IM、GitLab、Jenkins、禅道等十几个网页间来回切换，机械化流转耗费了 30% 以上的工作时间。
2. **传统自动化脚本的僵化**：现有的 CI/CD 脚本缺乏“上下文理解能力”，遇到特殊异常（如编译报错类型变异）只能阻断，需要人工介入。
3. **市面开源 Agent 的安全性灾难**：银行内网极度看重合规，市面上（如 LangChain/Dify 搭建的）Agent 平台缺乏“权限沙箱”与“人工防线”，大模型极易引发不可逆的生产事故（如错误删库、合错分支）。

## 🏗️ 2. 全景技术架构展示 (面向技术面试官的重火力)

在面试或技术评审中，面试官不仅看业务，更看重**架构设计的品味**。Ocean 的技术选型做到了“少即是多（Less is More）”，完美踩中了目前全栈开发与 AI 平台基建的最高阈值。

### 2.1 抛弃 LangChain，拥抱协议级解耦 (架构决策)
* **话术**：“市面上的低代码 Agent 平台在金融内网中往往水土不服。我之所以抛弃沉重的 LangChain，是因为在企业级编排中，**可控性**大于一切。我采用 **TypeScript DDD Monorepo** 架构，将外部基建（GitLab, Jenkins, Zentao）全部封装为独立的 **MCP Server**。网关层只负责协议分发，使得核心大脑代码实现了真正的‘高内聚、低耦合’。”

### 2.2 Ocean 独创的 AGP (Agent Governance Protocol)
* **面试杀手锏**：AGP 协议是系统的灵魂。
  * **动态沙盒白名单**：模型每次执行任务前，AGP 会根据业务意图注入**最小权限工具集**（例如只查询日志，绝不注入写代码工具）。
  * **HITL (Human-in-the-loop 人工介入断点)**：当大模型规划出高危操作（如 `merge_branch`, `trigger_build`）时，AGP 会挂起执行流，通过三栖终端（Web/IM/CLI）向用户下发确认卡片。**AI 只出方案，人类永远拥有最终扣动扳机的权力。**

### 2.3 极致的底层技术栈矩阵
* **生成式交互 (Generative UI)**：在 Web 端（React 18 + Vite），抛弃枯燥的 Markdown 文本流，接入 **Vercel AI SDK**。当查询 DevOps 数据时，系统直接流式渲染出可交互的 ECharts 报表组件。
* **高并发核心大脑**：后端网关摒弃了传统的 Python 脚本，采用 **NestJS + Node.js**，完美承载长耗时的 SSE（Server-Sent Events）连接与并发事件驱动。
* **统一记忆与向量搜索**：利用 **PostgreSQL (pgvector)** 统一处理业务数据与知识库高维向量计算（Cosine Distance）；利用 **Redis** 管理多轮对话的长短期记忆。
* **可观测性建设 (LLMOps)**：全链路接入 **OpenTelemetry (Jaeger)**。大模型的每一次思考耗时、Token 花费、工具调用的网络延迟都历历在目。

## 💼 3. 商业价值与落地推介 (面向高管/老板/决策层)

对管理层推介，核心是：**降本增效 (ROI) + 安全合规 (Compliance)**。

### 3.1 研发效能的指数级飞跃
* **话术**：“传统的自动化只能解决‘从 1 到 10’的过程，而 Ocean 解决了‘从 0 到 1 的分析与流转’。系统上线后，研发人员处理单个线上缺陷的平均耗时（分析日志 -> 禅道建 Bug -> 寻找负责人 -> 指派），从人工来回切系统的 40 分钟，缩短至一句话交互的 **5 分钟**。”
* **研发资产盘活**：通过自然语言查询，管理层可随时通过 Ocean 生成研发效能大盘卡片，所有隐性工作量（如排错时间）全部数据化。

### 3.2 金融级安全风控保障
* **话术**：“很多老板担心 AI 应用在内网会‘搞破坏’。Ocean 从架构设计的第一天就定下了金融级安全底线：**1. 严格绑定的 SSO 权限透传**（AI 操作就是本人操作，审计链条清晰）；**2. 所有高危变更必须人类盖章确认**（绝无 AI 越权可能）；**3. 本地化脱敏调度**（AI 仅接触意图，不存储敏感明文）。这是一套完全符合银行合规审查的系统。”

## 🤝 4. 消除阻力：内部平铺推广方案 (面向研发同事/测试团队)

内部推广的核心是：**消灭脏活累活、降低学习成本、绝不背锅**。

### 4.1 场景切入：直击最痛的“脏活”
* **致研发**：“还在开三个浏览器窗口找昨天的编译错误吗？在终端敲一个 `ocean fix`，它会自动读取你的本地代码，去 Jenkins 拉下错误日志，在终端直接输出代码修改方案。”
* **致测试/运维**：“发版还要填十几个参数表单？在企微群里艾特 Ocean：‘帮我把交易服务回退到 v1.2，并通知开发群’，Ocean 会弹出确认卡片，你点个【同意】就能去喝咖啡了。”

### 4.2 三栖触达，零门槛上手
* “我们没有强迫大家去学习一个新的网站。习惯命令行的极客直接用 **CLI**；习惯看大盘的去 **Web 面板**；出差或者下班在路上的，直接在 **企微/钉钉群** 艾特机器人。你的工作习惯完全不需要改变。”

## 📈 5. Ocean 的未来演进蓝图 (Roadmap)

在展示产品时，给出未来规划，彰显你的大局观。

* **Phase 1 (MVP 跑通 - 当前阶段)**：完成 MCP 协议底层打通，在小规模极客团队（种子用户）跑通“缺陷流转”与“CI/CD 触发”工作流。
* **Phase 2 (场景扩张)**：全面铺开至项目管理层，接入更多内网基建（如 JIRA, Confluence, 自动化测试平台），打造“研发效能度量”大模型驾驶舱。
* **Phase 3 (群智涌现)**：引入 Multi-Agent（多智能体）架构，代码评审 Agent、缺陷分析 Agent、发版管控 Agent 相互协作辩论，实现真正意义上的无人值守 DevOps 闭环。
